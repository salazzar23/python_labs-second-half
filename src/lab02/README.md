## Описание работы

В данной лабораторной работе реализован контейнерный класс `PatientCollection` для управления группой объектов предметной области "Медицина" (класс `Patient` из ЛР-1).

### Цель работы
- Научиться работать с коллекциями объектов
- Понять разницу между моделью сущности и контейнером объектов
- Реализовать собственный контейнерный класс
- Освоить итерацию по объектам
- Реализовать базовые операции управления коллекцией

---

## Предметная область: Медицина

### Сущность Patient (Пациент)

| Атрибут | Тип | Описание |
|---------|-----|----------|
| patient_id | str | Уникальный идентификатор пациента |
| name | str | Полное имя пациента |
| age | int | Возраст (1-149 лет) |
| diagnosis | str | Диагноз |
| last_appointment_date | datetime | Дата последнего приёма |
| doctor_specialization | str | Специализация врача |

### Методы класса Patient

| Метод | Описание |
|-------|----------|
| is_senior() | Проверка, является ли пациент пожилым (старше 65 лет) |
| needs_urgent_care() | Проверка необходимости срочной помощи |
| update_diagnosis() | Обновление диагноза |
| update_last_appointment() | Обновление даты последнего приёма |

## Реализованная функциональность

### Оценка 3 (Базовые операции)

| Метод | Описание |
|-------|----------|
| `add(item)` | Добавление объекта с проверкой типа |
| `remove(item)` | Удаление объекта |
| `get_all()` | Получение списка всех объектов |
| `remove_at(index)` | Удаление по индексу |

**Особенности:**
- Проверка типа добавляемых объектов
- Защита от добавления объектов неправильного типа
- Корректное удаление элементов

### Оценка 4 (Расширенная функциональность)

| Метод | Описание |
|-------|----------|
| `find_by_id()` | Поиск по ID пациента |
| `find_by_name()` | Поиск по имени (частичное совпадение) |
| `find_by_diagnosis()` | Поиск по диагнозу |
| `find_by_doctor_specialization()` | Поиск по специализации врача |
| `__len__()` | Поддержка функции len() |
| `__iter__()` | Поддержка итерации через for |

**Особенности:**
- Защита от добавления дубликатов (по patient_id)
- Все методы поиска возвращают списки найденных объектов

### Оценка 5 (Продвинутая функциональность)

| Метод | Описание |
|-------|----------|
| `__getitem__()` | Поддержка индексации и срезов |
| `sort(key)` | Универсальная сортировка |
| `sort_by_age()` | Сортировка по возрасту |
| `sort_by_name()` | Сортировка по имени |
| `sort_by_last_appointment()` | Сортировка по дате приёма |
| `filter(predicate)` | Универсальная фильтрация |
| `get_seniors()` | Получение пожилых пациентов |
| `get_urgent_care()` | Получение пациентов, требующих срочной помощи |
| `get_by_age_range()` | Получение пациентов в возрастном диапазоне |

**Особенности:**
- Поддержка отрицательных индексов (collection[-1])
- Поддержка срезов (collection[0:3])
- Методы сортировки и фильтрации возвращают новые коллекции
- Исходная коллекция не изменяется
